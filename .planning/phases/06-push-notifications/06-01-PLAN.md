   - Configure in OnModelCreating: primary keys, index on DeviceToken.CustomerId, string lengths (Token max 200, Platform max 10)
   - Connection string from appsettings.json: "IntegrationDb"

4. Create FcmNotificationService (src/backend/IntegrationService.Infrastructure/Services/FcmNotificationService.cs):
   - Implements INotificationService
   - Constructor inject: ILogger, IntegrationDbContext, IConfiguration
   - SendNotificationAsync(customerId, title, body, data):
     * Query DeviceToken WHERE CustomerId = customerId AND IsActive = true
     * For each token, create FCM Message with Token, Notification, Data
     * Android: Priority.High, ChannelId = "order_updates"
     * iOS: apns-priority = "5" (high priority)
     * Retry logic: 3 attempts with exponential backoff (1s, 3s, 9s)
     * On FirebaseMessagingException.Unregistered or InvalidArgument: mark token IsActive = false (don't retry invalid tokens)
     * On success: update DeviceToken.LastActive = DateTime.UtcNow
     * Log all attempts to NotificationLog (success or failure)
   - SendBroadcastNotificationAsync: throw NotImplementedException("Marketing campaigns deferred to M4")

5. Add EF migration:

   - Apply: dotnet ef database update --context IntegrationDbContext --startup-project ../IntegrationService.API/IntegrationService.API.csproj
