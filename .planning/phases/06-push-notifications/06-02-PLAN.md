  <action>
Create OrderStatusPollingService as IHostedService background worker to poll tblSales for TransType=9 orders and send ready notifications.

POLLING INTERVAL: 2 minutes (discretionary choice balancing DB load vs notification speed, as specified in 06-CONTEXT.md).

1. Create OrderStatusPollingService (src/backend/IntegrationService.Infrastructure/Services/OrderStatusPollingService.cs):
   - Inherit from BackgroundService (Microsoft.Extensions.Hosting)
   - Constructor inject: IServiceProvider, ILogger<OrderStatusPollingService>

2. Override ExecuteAsync(CancellationToken stoppingToken):
   ```csharp
   protected override async Task ExecuteAsync(CancellationToken stoppingToken)
   {
       _logger.LogInformation("Order Status Polling Service started with 2-minute interval (discretionary choice).");

       using PeriodicTimer timer = new(TimeSpan.FromMinutes(2));

       try
       {
           while (await timer.WaitForNextTickAsync(stoppingToken))
           {
               await PollOrderStatusAsync(stoppingToken);
           }
       }
       catch (OperationCanceledException)
       {
           _logger.LogInformation("Order Status Polling Service stopping.");
       }
       catch (Exception ex)
       {
           _logger.LogError(ex, "Order Status Polling Service encountered error.");
       }
   }
   ```

3. Implement PollOrderStatusAsync(CancellationToken stoppingToken):
   - Create service scope: using var scope = _services.CreateScope()
   - Resolve scoped dependencies:
     * var posRepo = scope.ServiceProvider.GetRequiredService<IPosRepository>()
     * var integrationDb = scope.ServiceProvider.GetRequiredService<IntegrationDbContext>()
     * var notificationService = scope.ServiceProvider.GetRequiredService<INotificationService>()
   - Query completed orders: var completedOrders = await posRepo.GetCompletedOnlineOrdersAsync()
   - For each completed order:
     * Check OnlineOrderStatus: var status = await integrationDb.OnlineOrderStatuses.FirstOrDefaultAsync(s => s.SalesId == order.ID)
     * If status == null: create new OnlineOrderStatus with ReadyNotificationSent = false
     * If status.ReadyNotificationSent == true: skip (already notified)
     * If status.ReadyNotificationSent == false:
       - Send notification:
         ```csharp
         await notificationService.SendNotificationAsync(
             order.CustomerID.Value,
             "Order ready for pickup!",
             $"Order #{order.DailyOrderNumber} is ready.",
             new Dictionary<string, string> {
                 { "type", "order_ready" },
                 { "orderId", order.ID.ToString() },
                 { "screen", "OrderTracking" }
             }
         );
         ```
       - Mark as sent: status.ReadyNotificationSent = true; status.LastCheckedAt = DateTime.UtcNow;
       - Save changes: await integrationDb.SaveChangesAsync(stoppingToken);
   - Graceful error handling: Wrap each order notification in try-catch (one failure shouldn't stop processing other orders)
  </action>

  <name>Task 4: Token and Log Cleanup Job</name>
  <files>
src/backend/IntegrationService.Infrastructure/Services/OrderStatusPollingService.cs
  </files>
  <action>
Extend OrderStatusPollingService with cleanup job to remove inactive tokens and old notification logs.

1. Add private field: private bool _firstRun = true;

2. In ExecuteAsync, run cleanup on first tick:
   ```csharp
   while (await timer.WaitForNextTickAsync(stoppingToken))
   {
       if (_firstRun)
       {
           await CleanupInactiveTokensAsync(stoppingToken);
           _firstRun = false;
       }
       await PollOrderStatusAsync(stoppingToken);
   }
   ```

3. Implement CleanupInactiveTokensAsync(CancellationToken stoppingToken):
   - Create service scope
   - Resolve IntegrationDbContext
   - Delete inactive tokens older than 30 days:
     ```csharp
     var cutoffDate = DateTime.UtcNow.AddDays(-30);
     var staleTokens = await integrationDb.DeviceTokens
         .Where(dt => !dt.IsActive && dt.LastActive < cutoffDate)
         .ToListAsync(stoppingToken);

     if (staleTokens.Any())
     {
         integrationDb.DeviceTokens.RemoveRange(staleTokens);
         await integrationDb.SaveChangesAsync(stoppingToken);
         _logger.LogInformation("Cleaned up {Count} stale device tokens (inactive >30 days).", staleTokens.Count);
     }
     ```

4. Add NotificationLog cleanup (90-day retention):
   ```csharp
   var logCutoff = DateTime.UtcNow.AddDays(-90);
   var oldLogs = await integrationDb.NotificationLogs
       .Where(nl => nl.CreatedAt < logCutoff)
       .ToListAsync(stoppingToken);

   if (oldLogs.Any())
   {
       integrationDb.NotificationLogs.RemoveRange(oldLogs);
       await integrationDb.SaveChangesAsync(stoppingToken);
       _logger.LogInformation("Cleaned up {Count} old notification logs (>90 days).", oldLogs.Count);
   }
   ```

Cleanup runs once on service startup, preventing database bloat from stale tokens and old logs.
